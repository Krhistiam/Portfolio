2. Automatización y procesamiento ETL de noticias en la web

Tecnologías aplicadas: Python, Web Scraping, Selenium, BeautifulSoup, Power Query, Lenguaje M.
▼

Este proyecto implementa un proceso de Extracción, Transformación y Carga (ETL) para recopilar y estructurar noticias desde un sitio web, facilitando su análisis y uso por analistas de datos y otros interesados.

Extracción de Datos:

  • Uso de Selenium y BeautifulSoup para hacer web scraping desde el portal web.
  • Captura de información clave: título de la noticia, fecha y enlace.
  • Almacenamiento en un archivo TXT para facilitar su procesamiento posterior.

Transformación de Datos:

  • Carga del archivo TXT en Power Query.
  • Conversión y limpieza de datos: Incluye estandarización del formato de fecha, conversión de texto a tabla estructurada y eliminación de datos duplicados o incompletos.

Impacto:

  • Automatización del proceso de recopilación de noticias.
  • Reducción del tiempo de búsqueda y estructuración de información.
  • Facilitación del análisis de identificación de frases y palabras claves.

from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from bs4 import BeautifulSoup
from time import gmtime, strftime
import time

url = "https://paginawebparascraping.com/"

# Configuración del driver
driver = webdriver.Chrome()
driver.get(url)

# Esperar que la página cargue completamente
time.sleep(10)

# Realizar scroll hacia abajo
scroll_pause_time = 5  # Tiempo de espera entre scrolls

# Hacer desplazamientos hacia abajo
for _ in range(20):
    # Hacer scroll hacia abajo
    driver.execute_script("window.scrollTo(0, document.body.scrollHeight);")
    
    # Esperar para que se carguen nuevos elementos
    time.sleep(scroll_pause_time)

# Extracción de info
soup = BeautifulSoup(driver.page_source, "html.parser")
articles = soup.find_all("article")
print(f"Found {len(articles)} articles")  # Confirmar cantidad de artículos

scraped_data = []
for article in articles:
    # Extraer los datos
    link_tag = article.find("a", class_="bodytext content")
    link = link_tag["href"] if link_tag else "No link found"

    title_tag = article.find("h2")
    title = title_tag.get_text(strip=True) if title_tag else "No title found"

    date_tag = article.find("time")
    date = date_tag["datetime"] if date_tag and date_tag.has_attr("datetime") else "No date found"

    # Imprimir datos para depuración
    print(f"Link: {link}")
    print(f"Title: {title}")
    print(f"Date: {date}")

    # Agregar a la lista solo si todos los datos están disponibles
    if link != "No link found" and title != "No title found" and date != "No date found":
        scraped_data.append({"title": title, "link": link, "date": date})

# Guardar los datos en un archivo .txt
fecha = strftime("%Y-%m-%d %H-%M-%S", gmtime())
file_name = f"cision_scraped_data_{fecha}.txt"

with open(file_name, "w", encoding="utf-8") as file:
    for entry in scraped_data:
        file.write(f"Title: {entry['title']}\n")
        file.write(f"Date: {entry['date']}\n")
        file.write(f"Link: {entry['link']}\n")
        file.write("\n")  # Línea vacía entre noticias

# Mensaje final
print(f"Datos guardados correctamente en '{file_name}'")

# Cerrar el navegador
driver.quit()

3. Análisis y Segmentación de Vinos con Machine Learning

Tecnologías aplicadas: Python, Pandas, Scikit-Learn, Matplotlib, numpy.
▼

Este proyecto aplica técnicas de aprendizaje automático para analizar y segmentar diferentes variedades de vino a partir de sus características químicas. Utilizando el dataset Wine Recognition de Scikit-Learn, se implementa un flujo de trabajo basado en reducción de dimensionalidad y clustering para identificar patrones en los datos sin necesidad de etiquetas predefinidas.

Metodología:

  • Normalización de Datos: Los datos son transformados restando la media y dividiendo por la desviación estándar para asegurar comparabilidad.
  • Reducción de Dimensionalidad: Se aplica Análisis de Componentes Principales (PCA) para reducir la dimensionalidad a dos componentes principales y visualizar los datos en un gráfico de dispersión.
  • División del Conjunto de Datos: Se separan los datos en entrenamiento y prueba usando train_test_split con una proporción entre 20% y 30% para evaluación.
  • Clustering con K-Means: Se prueba el algoritmo K-Means con distintos valores de K (2 a 5) y se evalúa el rendimiento con Adjusted Rand Score.
  • Clustering con Propagación de Afinidad: Se evalúa el algoritmo Affinity Propagation para segmentar los datos sin etiquetas y se mide la precisión obtenida, buscando alcanzar al menos un 90%.
  • Interpretación de Resultados: Se analizan las diferencias entre los métodos de clustering y la efectividad de PCA en la reducción de dimensionalidad.

Impacto:

  • Preprocesamiento de datos, reducción de dimensionalidad y clustering no supervisado.
  • Extración de patrones en conjuntos de datos sin etiquetas, aplicable a diversos sectores como segmentación de clientes, análisis de productos entre otros.

Resultado:

  • Tras evaluar el desempeño del algoritmo K-Means con diferentes valores de K, se observó que el mejor resultado de Adjusted Rand Score (ARS) se obtuvo con K=3, alcanzando un valor de 0.93. Este resultado es consistente con la naturaleza del conjunto de datos, ya que el dataset contiene etiquetas reales de tres clases de vinos. Con K=4, el ARS disminuyó a 0.76, lo que indica que el modelo estaba forzando la división de una de las tres clases originales en dos clústeres, afectando la correspondencia con las categorías reales. Para K=5, el rendimiento se redujo aún más debido a una segmentación excesiva en clases innecesarias. Con K=2, el desempeño fue el más bajo, dado que agrupar los vinos en solo dos categorías no representa adecuadamente la variabilidad del dataset. En cuanto a la aplicación del algoritmo de propagación de afinidad, se obtuvo un ARS de 0.93, cumpliendo con la condición establecida de superar el 90% de precisión. Esto indica que el modelo logró identificar patrones en los datos alineándose con las categorías reales del conjunto de datos, a pesar de tratarse de un modelo de aprendizaje no supervisado, donde las etiquetas originales no fueron utilizadas en el proceso de entrenamiento. Estos resultados demuestran la eficacia de las técnicas de clustering para segmentar datos sin etiquetas y refuerzan la importancia de seleccionar el número óptimo de clústeres en problemas de clasificación no supervisada.
